# Generated and Manual File Boundary

`pygodot` is source-driven. The Python project is the source of truth; the
Godot project under `build_dir` is generated output.

## Source root

`Game.source_root` points at user-owned source files:

- Python files that declare `Game`, `Scene`, nodes, scripts, input actions, and
  resources;
- manually authored assets;
- manually authored GDScript files referenced with `Script.reference(...)`;
- other project files the user edits directly.

Files under `source_root` are not overwritten by `Game.build()`.

## Build directory

`Game.build_dir` contains the generated Godot project:

```text
build/godot_project/
  .pygodot/
    manifest.json
  project.godot
  scenes/
  scripts/
  assets/
```

Files written by emitters in `build_dir` may be overwritten on every build.
Do not treat generated `.tscn`, `.gd`, `.tres`, `project.godot`, or
`.pygodot/manifest.json` files as the source of truth.

## Generated files

Generated files are owned by pygodot when they are emitted from the Python DSL
or generated GDScript bodies:

- `project.godot`;
- generated scene files such as `res://scenes/main.tscn`;
- generated scripts from `Script(path=..., body=...)`;
- generated scripts from `Script.from_file(source=..., path=..., extends=...)`;
- generated scripts from
  `Script.from_template(source=..., path=..., extends=..., context=...)`;
- future generated resources such as `.tres`;
- `.pygodot/manifest.json`.

Generated text files include a `Generated by pygodot` marker where the file type
supports a comment header. The generated file writer uses this marker to avoid
silently overwriting manual files when overwrite protection is enabled.

## Manual scripts

`Script.from_file(...)` and `Script.from_template(...)` are still generated
output. Their source files are user-owned input under `source_root`, and
pygodot writes the generated `.gd` file under `build_dir`.

Manual scripts should be referenced explicitly:

```python
Script.reference("res://manual/player.gd", extends="CharacterBody2D")
```

Referenced manual scripts are included in generated `.tscn` files as external
script resources, but `Game.build()` does not write the script file. The user is
responsible for keeping the manual `.gd` file in `source_root` or otherwise
available to the generated Godot project.

Do not pass a body to `Script.reference(...)`. A script with a body is treated as
generated and belongs to pygodot.

## External resources

External resources declared with helpers such as `texture(...)`,
`audio_stream(...)`, `font(...)`, `packed_scene(...)`, or `ext_resource(...)` are
source-owned resources.

During a build, if the resource path exists under `source_root`, pygodot copies
it to the same `res://` relative path under `build_dir`. If the file does not
exist, the generated project still references it and the manifest records it
with `copied=false`.

Copied resources are not generated content. Edit the source-owned file under
`source_root`, then rebuild.

## Manifest

Each build writes:

```text
res://.pygodot/manifest.json
```

The manifest records:

- `generated_files`: generated project files written by pygodot;
- `generated_scenes`: generated `.tscn` files;
- `generated_scripts`: generated `.gd` files;
- `external_resources`: external resources, their stable IDs, and whether they
  were copied from `source_root`.

The manifest is generated metadata. It helps inspect ownership and build output,
but it is not the source of truth for the game.

## Practical rule

Edit Python DSL files, manual assets, and explicitly referenced manual scripts.
Rebuild instead of editing generated output by hand.
